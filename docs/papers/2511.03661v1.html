SHIELD: Securing Healthcare IoT with Efficient Machine Learning Techniques for Anomaly Detection

Published: 2025-11-05
Authors: Mahek Desai, Apoorva Rumale, Marjan Asadinia
Categories: cs.LG

Summary

This study proposes SHIELD, a machine learning framework designed to detect both malicious cyberattacks and faulty device anomalies in healthcare IoT environments. By evaluating eight diverse machine learning models on a 200,000-record dataset, the research identifies highly efficient and accurate techniques for enhancing IoT-enabled healthcare security. The findings highlight XGBoost and KNN as top performers for anomaly and attack detection respectively, offering significant potential for safeguarding patient health and data.

Medical Relevance

This research is paramount for ensuring the security, reliability, and continuous operation of medical IoT devices, directly impacting patient safety, data privacy, and trust in technology-driven healthcare delivery. Effective anomaly and attack detection can prevent critical disruptions and protect sensitive health information.

AI Health Application

The AI application involves using various machine learning models (e.g., XGBoost, K-NN, GAN, VAE, SVM, Isolation Forest, GNN, LSTM Autoencoders) for anomaly detection to identify malicious cyberattacks and faulty device operations within healthcare IoT ecosystems. This application aims to improve the security, reliability, and continuous safe operation of medical devices and systems, thereby protecting patient data and well-being.

Key Points

  • Addresses critical security and reliability challenges in healthcare IoT, focusing on cyber threats and operational device anomalies.
  • Proposes a machine learning-driven framework (SHIELD) to achieve two distinct goals: detecting malicious cyberattacks and identifying faulty device anomalies.
  • Evaluated eight diverse machine learning models across supervised (XGBoost, K-NN), semi-supervised (GAN, VAE), and unsupervised (One-Class SVM, Isolation Forest, GNN, LSTM Autoencoders) learning approaches.
  • Utilized a substantial dataset of 200,000 records for comprehensive model evaluation using metrics such as F1-score, precision, recall, accuracy, ROC-AUC, and computational efficiency.
  • For anomaly detection, XGBoost achieved 99% accuracy with minimal computational overhead (0.04s), while Isolation Forest demonstrated an effective balance of precision and recall.
  • For cyberattack detection, K-Nearest Neighbors (KNN) achieved near-perfect precision, recall, and F1-score with the lowest computational cost (0.05s), followed by VAE at 97% accuracy.
  • The framework's implementation can significantly improve early detection of threats and failures, thereby preventing data breaches, minimizing system downtime, and ensuring the continuous, safe operation of medical devices.

Methodology

The study employed a comparative analysis of eight machine learning models across supervised (XGBoost, K-NN), semi-supervised (GAN, VAE), and unsupervised (One-Class SVM, Isolation Forest, GNN, LSTM Autoencoders) learning paradigms. These models were trained and evaluated on a dataset of 200,000 records for two tasks: cyberattack detection and faulty device anomaly identification. Performance was assessed using a comprehensive suite of metrics including F1-score, precision, recall, accuracy, ROC-AUC, and computational efficiency.

Key Findings

For anomaly detection, XGBoost achieved the highest accuracy of 99% with the lowest computational overhead (0.04s), while Isolation Forest balanced precision and recall effectively. LSTM Autoencoders underperformed with lower accuracy and higher latency. For attack detection, K-Nearest Neighbors (KNN) achieved near-perfect precision, recall, and F1-score with the lowest computational cost (0.05s). Variational Autoencoders (VAE) followed with 97% accuracy, whereas Generative Adversarial Networks (GAN) showed the highest computational cost and lowest accuracy/ROC-AUC.

Clinical Impact

Implementing the SHIELD framework in clinical settings can significantly enhance the security posture of healthcare IoT infrastructure. It enables early and efficient detection of both cyber threats and device malfunctions, directly preventing costly data breaches, minimizing system downtime for critical medical equipment, and ensuring the uninterrupted, safe operation of devices. This ultimately safeguards patient health, maintains trust in IoT-driven healthcare solutions, and supports more resilient healthcare operations.

Abstract

The integration of IoT devices in healthcare introduces significant security and reliability challenges, increasing susceptibility to cyber threats and operational anomalies. This study proposes a machine learning-driven framework for (1) detecting malicious cyberattacks and (2) identifying faulty device anomalies, leveraging a dataset of 200,000 records. Eight machine learning models are evaluated across three learning approaches: supervised learning (XGBoost, K-Nearest Neighbors (K- NN)), semi-supervised learning (Generative Adversarial Networks (GAN), Variational Autoencoders (VAE)), and unsupervised learning (One-Class Support Vector Machine (SVM), Isolation Forest, Graph Neural Networks (GNN), and Long Short-Term Memory (LSTM) Autoencoders). The comprehensive evaluation was conducted across multiple metrics like F1-score, precision, recall, accuracy, ROC-AUC, computational efficiency. XGBoost achieved 99\% accuracy with minimal computational overhead (0.04s) for anomaly detection, while Isolation Forest balanced precision and recall effectively. LSTM Autoencoders underperformed with lower accuracy and higher latency. For attack detection, KNN achieved near-perfect precision, recall, and F1-score with the lowest computational cost (0.05s), followed by VAE at 97% accuracy. GAN showed the highest computational cost with lowest accuracy and ROC-AUC. These findings enhance IoT-enabled healthcare security through effective anomaly detection strategies. By improving early detection of cyber threats and device failures, this framework has the potential to prevent data breaches, minimize system downtime, and ensure the continuous and safe operation of medical devices, ultimately safeguarding patient health and trust in IoT-driven healthcare solutions.
