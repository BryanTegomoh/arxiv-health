UnifiedFL: A Dynamic Unified Learning Framework for Equitable Federation

arXiv ID: 2510.26350v1
Published: 2025-10-30
Authors: Furkan Pala, Islem Rekik
Categories: cs.LG
Relevance Score: 0.95 / 1.00
View on arXiv Download PDF

Summary

UnifiedFL is a novel federated learning framework designed to overcome significant challenges in collaborative model training, namely architectural heterogeneity (clients using fundamentally different network types) and the 'domain-fracture problem' where test data distributions differ markedly from training data. It achieves this by representing diverse local networks within a directed model graph, parameterized by a shared Graph Neural Network (GNN), and employing a unique clustering and two-tier aggregation policy. Experiments on MedMNIST classification and hippocampus segmentation benchmarks demonstrate its superior performance.

Medical Relevance

This framework is highly relevant for medical/health applications as it enables privacy-preserving collaborative AI development across diverse healthcare institutions. It addresses the practical reality of varied technological infrastructures and patient populations, allowing for the creation of more robust and generalizable models in critical areas like radiology and pathology without sharing sensitive raw patient data.

AI Health Application

UnifiedFL is an advanced federated learning framework that facilitates the collaborative training of AI models across multiple healthcare providers. It specifically addresses challenges inherent in real-world medical settings, such as the use of fundamentally different neural architectures (e.g., CNNs, GNNs, MLPs) and non-identically distributed patient datasets across institutions. By enabling more robust and generalizable AI models for tasks like medical image classification (e.g., MedMNIST) and anatomical segmentation (e.g., hippocampus), while preserving data privacy, UnifiedFL directly supports the development and deployment of more effective and ethical AI solutions in clinical diagnostics and treatment planning.

Key Points

  • Addresses critical limitations in current federated learning (FL) frameworks, including the inability to accommodate fundamentally different neural architectures (e.g., CNNs, GNNs, MLPs) beyond mere structural variants.
  • Tackles the 'domain-fracture problem' where each client's data distribution differs significantly from the eventual testing domain, hindering model generalizability.
  • Introduces UnifiedFL, a dynamic FL framework that represents heterogeneous local networks as nodes and edges in a directed model graph.
  • Employs a common Graph Neural Network (GNN) to parameterize and unify all diverse client architectures within the federated network.
  • Incorporates a distance-driven clustering mechanism based on Euclidean distances between client parameters to group similar models, aiding in tailored aggregation.
  • Proposes a two-tier aggregation policy designed to balance the trade-off between model convergence speed and maintaining diversity across the federated models.
  • Validated on important medical imaging benchmarks: MedMNIST classification and hippocampus segmentation, showcasing superior performance under high architectural and statistical heterogeneity.

Methodology

UnifiedFL represents heterogeneous local neural networks (e.g., CNNs, GNNs, MLPs) as nodes and edges within a directed model graph. A common Graph Neural Network (GNN) is utilized to parameterize all these diverse architectures. The framework incorporates distance-driven clustering, grouping clients based on Euclidean distances between their model parameters. Finally, it employs a two-tier aggregation policy designed to effectively balance model convergence and maintain diversity across the federated network.

Key Findings

UnifiedFL demonstrates superior performance compared to existing federated learning methods. This was evidenced in experiments on MedMNIST classification and hippocampus segmentation benchmarks, particularly under challenging conditions involving fundamentally different client network architectures, non-identically distributed datasets, and distinct test domains, thus addressing critical limitations in model generalizability.

Clinical Impact

UnifiedFL's ability to seamlessly integrate models from diverse architectural types and handle varied data distributions significantly lowers barriers for collaborative AI development in healthcare. This can lead to more generalizable and robust diagnostic and prognostic models, directly improving patient care by leveraging data from a wider range of hospitals and research centers while upholding data privacy in fields like cancer detection (pathology) and neurological disease analysis (radiology).

Abstract

Federated learning (FL) has emerged as a key paradigm for collaborative model training across multiple clients without sharing raw data, enabling privacy-preserving applications in areas such as radiology and pathology. However, works on collaborative training across clients with fundamentally different neural architectures and non-identically distributed datasets remain scarce. Existing FL frameworks face several limitations. Despite claiming to support architectural heterogeneity, most recent FL methods only tolerate variants within a single model family (e.g., shallower, deeper, or wider CNNs), still presuming a shared global architecture and failing to accommodate federations where clients deploy fundamentally different network types (e.g., CNNs, GNNs, MLPs). Moreover, existing approaches often address only statistical heterogeneity while overlooking the domain-fracture problem, where each client's data distribution differs markedly from that faced at testing time, undermining model generalizability. When clients use different architectures, have non-identically distributed data, and encounter distinct test domains, current methods perform poorly. To address these challenges, we propose UnifiedFL, a dynamic federated learning framework that represents heterogeneous local networks as nodes and edges in a directed model graph optimized by a shared graph neural network (GNN). UnifiedFL introduces (i) a common GNN to parameterize all architectures, (ii) distance-driven clustering via Euclidean distances between clients' parameters, and (iii) a two-tier aggregation policy balancing convergence and diversity. Experiments on MedMNIST classification and hippocampus segmentation benchmarks demonstrate UnifiedFL's superior performance. Code and data: https://github.com/basiralab/UnifiedFL
