RELATE: A Schema-Agnostic Perceiver Encoder for Multimodal Relational Graphs

arXiv ID: 2510.19954v1
Published: 2025-10-22
Authors: Joseph Meyer, Divyansha Lachi, Reza Mohammadi, Roshan Reddy Upendra, Eva L. Dyer, Mark Li, Tom Palczewski
Categories: cs.AI, cs.DB, cs.LG
Relevance Score: 0.90 / 1.00
View on arXiv Download PDF

Summary

This paper introduces RELATE, a novel schema-agnostic Perceiver encoder designed for multimodal relational graph data, common in domains like healthcare. By utilizing shared modality-specific encoders and a Perceiver-style cross-attention mechanism, RELATE generates fixed-size, permutation-invariant node representations, enabling plug-and-play integration with any general-purpose Graph Neural Network (GNN). Evaluations on the RelBench benchmark demonstrate that RELATE achieves performance within 3% of schema-specific encoders while significantly reducing parameter counts by up to 5x, facilitating multi-dataset pretraining and the development of foundation models for complex relational data.

Medical Relevance

Healthcare data is inherently complex, multimodal (e.g., EHRs, imaging, genomics, sensor data), and relational (patient histories, disease networks, drug interactions). RELATE offers a robust, efficient, and flexible method to integrate and process these diverse data types without requiring custom architectural modifications for every new dataset or schema, accelerating AI development in clinical informatics and research.

AI Health Application

RELATE provides a foundational technical advancement for medical AI by enabling more robust, scalable, and schema-agnostic graph neural networks. This can be applied to develop AI models that can analyze diverse and complex healthcare datasets (e.g., patient histories, lab results, imaging reports, genetic data) to: 1) improve personalized medicine by identifying optimal treatments based on patient profiles, 2) enhance clinical decision support by predicting disease progression or treatment response, 3) accelerate drug discovery by modeling complex interactions in biological systems, and 4) facilitate multi-institutional data integration and analysis without requiring extensive schema harmonization.

Key Points

  • **Problem Addressed:** Existing GNNs for relational multi-table data require schema-specific feature encoders, leading to separate modules for each node type and feature column, which hinders scalability and parameter sharing.
  • **Solution Proposed:** RELATE (Relational Encoder for Latent Aggregation of Typed Entities), a schema-agnostic, plug-and-play feature encoder.
  • **Architectural Innovation:** Employs shared modality-specific encoders for categorical, numerical, textual, and temporal attributes, followed by a Perceiver-style cross-attention module.
  • **Output:** Aggregates features into a fixed-size, permutation-invariant node representation, making it compatible with various GNN backbones.
  • **Evaluation:** Tested on ReLGNN and HGT architectures within the RelBench benchmark.
  • **Key Result 1:** Achieves performance within 3% of traditional schema-specific encoders, demonstrating strong feature representation capabilities.
  • **Key Result 2:** Significantly reduces parameter counts by up to 5x compared to schema-specific approaches, improving model efficiency and scalability.
  • **Future Implications:** Its design supports varying schemas and enables multi-dataset pretraining, paving the way for developing foundation models tailored for relational graph data.

Methodology

The RELATE architecture consists of two main components: 1) Shared modality-specific encoders, which process different data types (e.g., MLPs for numerical/temporal, embedding layers for categorical, Transformer-like modules for textual data). These encoders are shared across all node types and feature columns. 2) A Perceiver-style cross-attention module that takes the outputs of these modality-specific encoders and aggregates them into a fixed-size, permutation-invariant latent representation for each node. This unified node representation can then be fed into any general-purpose GNN (e.g., ReLGNN, HGT). The method was evaluated on benchmark tasks from RelBench, comparing performance (accuracy/AUC) and parameter efficiency against schema-specific encoder baselines.

Key Findings

The primary findings demonstrate RELATE's ability to maintain high performance while significantly enhancing model efficiency and flexibility. Specifically, it achieved performance within 3% of highly specialized, schema-specific encoders, indicating its robust feature extraction capabilities. Concurrently, it reduced the total parameter count by up to 5 times, highlighting its superior parameter efficiency. This signifies a breakthrough in handling heterogeneous relational data, making it feasible to train more generalized and scalable models.

Clinical Impact

RELATE could revolutionize how AI models are developed and deployed in healthcare. It would enable seamless integration of disparate and evolving healthcare data sources (e.g., merging EHRs from multiple hospitals with diverse structures, integrating novel '-omics' data). This reduces development costs and time, accelerates the creation of accurate predictive models for disease onset, treatment response, and patient risk stratification. Its potential for 'foundation models' means a single, large-scale model pre-trained on vast amounts of healthcare data could be fine-tuned for a multitude of specific clinical tasks, improving model robustness, generalizability, and applicability across various medical conditions and patient populations, even for rare diseases with limited individual datasets.

Limitations

The abstract does not explicitly state limitations. However, potential considerations for this type of architecture often include: 1) While efficient in parameter count, the computational cost of the Perceiver-style attention itself could be a factor for extremely large graphs or very long input sequences per node, although typically optimized for fixed-size outputs. 2) The 'within 3%' performance gap, while small and offset by parameter reduction, indicates a minor trade-off compared to highly specialized, schema-optimized encoders. 3) The effectiveness ultimately depends on the quality of the underlying GNN it's plugged into.

Future Directions

The paper explicitly states that this design 'paves the way toward foundation models for relational graph data.' This suggests future work will focus on extensive multi-dataset pretraining of RELATE-equipped GNNs to build highly generalizable models that can be fine-tuned for a wide array of downstream tasks across various domains, including complex healthcare applications.

Abstract

Relational multi-table data is common in domains such as e-commerce, healthcare, and scientific research, and can be naturally represented as heterogeneous temporal graphs with multi-modal node attributes. Existing graph neural networks (GNNs) rely on schema-specific feature encoders, requiring separate modules for each node type and feature column, which hinders scalability and parameter sharing. We introduce RELATE (Relational Encoder for Latent Aggregation of Typed Entities), a schema-agnostic, plug-and-play feature encoder that can be used with any general purpose GNN. RELATE employs shared modality-specific encoders for categorical, numerical, textual, and temporal attributes, followed by a Perceiver-style cross-attention module that aggregates features into a fixed-size, permutation-invariant node representation. We evaluate RELATE on ReLGNN and HGT in the RelBench benchmark, where it achieves performance within 3% of schema-specific encoders while reducing parameter counts by up to 5x. This design supports varying schemas and enables multi-dataset pretraining for general-purpose GNNs, paving the way toward foundation models for relational graph data.
