GRASP: Graph Reasoning Agents for Systems Pharmacology with Human-in-the-Loop

Summary

GRASP is a multi-agent, graph-reasoning framework with a human-in-the-loop conversational interface designed to automate and streamline Quantitative Systems Pharmacology (QSP) model development. It encodes QSP models as typed biological knowledge graphs, compiles them to executable MATLAB/SimBiology code while preserving critical constraints, and significantly outperforms baseline methods in terms of model quality and correctness. This framework makes complex QSP modeling more accessible and rigorous for domain experts, accelerating drug development.

Medical Relevance

Quantitative Systems Pharmacology (QSP) modeling is crucial for understanding drug mechanisms, predicting drug behavior, and optimizing drug development processes, which directly impacts the safety, efficacy, and speed of bringing new therapies to patients. GRASP automates and streamlines this complex modeling, accelerating pharmaceutical research and development.

Key Points

  • GRASP is a multi-agent, graph-reasoning framework that integrates a human-in-the-loop conversational interface for developing Quantitative Systems Pharmacology (QSP) models.
  • It encodes QSP models as typed biological knowledge graphs and compiles them into executable MATLAB/SimBiology code, rigorously preserving units, mass balance, and physiological constraints.
  • The framework operates through a two-phase workflow: 'Understanding' (graph reconstruction from legacy code) and 'Action' (constraint-checked, language-driven model modification), orchestrated by a state machine with iterative validation.
  • GRASP incorporates a breadth-first parameter-alignment (BFS alignment) mechanism to automatically discover dependencies, align parameters, and propose biologically plausible defaults for new entities, achieving an F1 score of 0.95 for these tasks.
  • In evaluations using an LLM-as-judge, GRASP achieved significantly higher scores (approximately 9-10/10) across biological plausibility, mathematical correctness, structural fidelity, and code quality compared to SME-guided CoT and ToT baselines (5-7/10).
  • The system allows domain experts to specify complex QSP mechanisms using natural language, drastically reducing the time investment typically required without sacrificing biomedical fidelity.
  • GRASP's ability to automate code generation and perform automatic execution/diagnostics until convergence ensures robust and validated QSP models.

Methodology

GRASP employs a multi-agent architecture built around typed biological knowledge graphs to represent QSP models. Its workflow includes an "Understanding" phase for reconstructing graphs from existing code and an "Action" phase for language-driven, constraint-checked model modification, orchestrated by a state machine with iterative validation. It utilizes breadth-first parameter-alignment (BFS alignment) to manage dependencies and parameter proposals, and automatically generates executable MATLAB/SimBiology code while ensuring physiological and mathematical constraints. Performance was assessed using an LLM-as-judge comparing GRASP against SME-guided Chain-of-Thought (CoT) and Tree-of-Thought (ToT) baselines.

Key Findings

GRASP demonstrated superior performance, scoring approximately 9-10/10 across critical metrics like biological plausibility, mathematical correctness, structural fidelity, and code quality, significantly outperforming SME-guided CoT and ToT baselines (5-7/10) as evaluated by an LLM-as-judge. Its BFS alignment mechanism achieved a high F1 score of 0.95 for dependency discovery, unit validation, and range checking. These findings indicate that GRASP's graph-structured, agentic workflow substantially enhances the rigor and accessibility of QSP model development.

Clinical Impact

By accelerating and improving the rigor of QSP model development, GRASP can significantly impact pharmaceutical research and development. It enables researchers to more rapidly design, simulate, and interpret complex pharmacological systems, leading to more efficient identification of drug targets, better prediction of drug responses, reduced need for costly preclinical and clinical trials, and ultimately, faster delivery of safer and more effective treatments to patients. Its accessibility to domain experts can democratize QSP modeling, fostering innovation across drug discovery.

Abstract

Quantitative Systems Pharmacology (QSP) modeling is essential for drug development but it requires significant time investment that limits the throughput of domain experts. We present \textbf{GRASP} -- a multi-agent, graph-reasoning framework with a human-in-the-loop conversational interface -- that encodes QSP models as typed biological knowledge graphs and compiles them to executable MATLAB/SimBiology code while preserving units, mass balance, and physiological constraints. A two-phase workflow -- \textsc{Understanding} (graph reconstruction of legacy code) and \textsc{Action} (constraint-checked, language-driven modification) -- is orchestrated by a state machine with iterative validation. GRASP performs breadth-first parameter-alignment around new entities to surface dependent quantities and propose biologically plausible defaults, and it runs automatic execution/diagnostics until convergence. In head-to-head evaluations using LLM-as-judge, GRASP outperforms SME-guided CoT and ToT baselines across biological plausibility, mathematical correctness, structural fidelity, and code quality (\(\approx\)9--10/10 vs.\ 5--7/10). BFS alignment achieves F1 = 0.95 for dependency discovery, units, and range. These results demonstrate that graph-structured, agentic workflows can make QSP model development both accessible and rigorous, enabling domain experts to specify mechanisms in natural language without sacrificing biomedical fidelity.
