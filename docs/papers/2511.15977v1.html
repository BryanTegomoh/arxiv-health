Summary

This paper addresses the challenges of high memory consumption and inefficient resource utilization in large-scale precision medicine genomic workflows by proposing adaptive, RAM-efficient parallelization mechanisms. It introduces symbolic and polynomial regression models for memory prediction, dynamic and static schedulers to optimize task packing and processing order, ultimately reducing memory overruns, balancing load, and accelerating end-to-end execution.

Medical Relevance

This work is highly relevant to precision medicine by providing crucial tools to efficiently process and analyze the large genomic datasets essential for personalized diagnostics, targeted therapies, and population-scale genetic research, thereby accelerating discoveries and clinical implementations.

AI Health Application

AI/Machine Learning models (symbolic and polynomial regression) are applied to predict per-chromosome memory consumption for genomic tasks. This enables dynamic and static scheduling optimization (treating task packing as a Knapsack problem) within precision medicine genomic workflows. The AI-driven optimization reduces memory overruns, balances load, and achieves faster end-to-end execution, directly improving the efficiency and scalability of genomic analysis critical for clinical applications in precision medicine.

Key Points

  • Large-scale genomic workflows, critical for precision medicine, frequently encounter high memory spikes, intensive disk I/O, and out-of-memory (OOM) errors due to the variable RAM demands of per-chromosome processing.
  • A symbolic regression model is developed to accurately estimate per-chromosome memory consumption for specific tasks, incorporating an interpolating bias to conservatively minimize memory over-allocation.
  • A dynamic scheduler is introduced that leverages a polynomial regression model for adaptive RAM usage prediction, framing task packing as a Knapsack problem to optimally batch jobs based on their predicted memory requirements.
  • A static scheduler is also presented, which optimizes the order of chromosome processing to minimize peak memory consumption across the workflow while preserving overall throughput.
  • Evaluation on simulations and real-world genomic pipelines demonstrates that the proposed methods effectively reduce memory overruns and improve load balancing across processing threads.
  • The adaptive parallelization strategies lead to significantly faster end-to-end execution times, showcasing their potential to optimize complex and resource-intensive genomic workflows.
  • These mechanisms enable more robust and efficient processing of genomic data, preventing costly failures and improving the reliability of precision medicine applications.

Methodology

The authors developed three primary mechanisms: (1) A symbolic regression model with an interpolating bias for conservative per-chromosome memory estimation. (2) A dynamic scheduler using a polynomial regression model for adaptive RAM usage prediction, framing task packing as a Knapsack problem for optimal batching. (3) A static scheduler to optimize chromosome processing order for peak memory minimization while preserving throughput. These methods were evaluated through simulations and real-world genomic pipelines.

Abstract

Large-scale genomic workflows used in precision medicine can process datasets spanning tens to hundreds of gigabytes per sample, leading to high memory spikes, intensive disk I/O, and task failures due to out-of-memory errors. Simple static resource allocation methods struggle to handle the variability in per-chromosome RAM demands, resulting in poor resource utilization and long runtimes. In this work, we propose multiple mechanisms for adaptive, RAM-efficient parallelization of chromosome-level bioinformatics workflows. First, we develop a symbolic regression model that estimates per-chromosome memory consumption for a given task and introduces an interpolating bias to conservatively minimize over-allocation. Second, we present a dynamic scheduler that adaptively predicts RAM usage with a polynomial regression model, treating task packing as a Knapsack problem to optimally batch jobs based on predicted memory requirements. Additionally, we present a static scheduler that optimizes chromosome processing order to minimize peak memory while preserving throughput. Our proposed methods, evaluated on simulations and real-world genomic pipelines, provide new mechanisms to reduce memory overruns and balance load across threads. We thereby achieve faster end-to-end execution, showcasing the potential to optimize large-scale genomic workflows.
